# Design Guide — Yoonbin's UI/UX Portfoliio 
> 이 문서는 이 프로젝트의 디자인 기준이다.
> 모든 화면과 컴포넌트는 이 문서를 먼저 읽고 따른다.
> 규칙과 어긋나면 만들기 전에 멈추고 나에게 물어본다.

## 1. Overview (브랜드 한 줄·무드)
- 브랜드명: UI/UX Portfoliio 
- 한 문장: visual storytelling and user-centered problem solving의 아이덴티티를 잘 보여줄 수 있는 프로덕트 디자인 포토폴리오
- 무드 키워드: Mimimal / 타입 중심 / 따뜻한 포인트 / Soft / Gradient
- 느낌 비교: labs.google 같은 깔끔한 에디토리얼 O · 알록달록 사탕색 X

## 2. Colors (컬러)
- Primary (포인트): #B0C6E1
- Secondary (서브 포인트): #98AD57
- Ink (제목·본문): #1a1a1a
- Muted (보조 텍스트): #7a7a72
- Surface (카드·띄): #F2F2F0
- Background (배경): #FAFAF8
- 규칙: 색은 위계에만. 포인트는 아이콘·브랜드명·버튼에만, 카드는 중립면.
- 의미 규칙 (전 케이스 공통): 블루 = 구조·섹션 라벨·인사이트 등 기본 강조 / 올리브 = 솔루션·결과·개선 방향 전용 (To, Strategy, TO-BE, Solutions, 긍정 지표). 두 색의 역할을 케이스마다 바꾸지 않는다.

## 3. Typography (타이포그래피)
- 폰트: Manrope (전부 이 하나만)
- H1 64px / 800 · H2 32px / 800 · H3 18px / 700 · 본문 16px / Regular
- 라벨 14px / 700 (#B0C6E1) · 행간 본문 1.6
- 규칙: 폰트는 하나. 위계는 크기·굵기로만.

## 4. Layout (간격·여백)
- 기준 단위: 8px 그리드 (8 / 16 / 24 / 32 / 48 / 64 / 96)
- 콘텐츠 최대 너비 1200px · 섹션 패딩 96px · 카드 간격 24px · 좌우 여백 48px
- 규칙: 위 단위 밖 어중간한 값 금지.

## 5. Elevation & Depth (그림자·깊이)
- 기본은 그림자 없음 — 면과 여백으로 구분한다.
- 호버 시에만 얙게: 0 16px 38px rgba(20,20,30,.08) + 살짝 떠오르기.
- 규칙: 라이트그레이 구분선으로 영역을 나누지 않는다.

## 6. Shapes (모서리·형태)
- 모서리: 카드 22px / CTA 패널 28px / 버튼 pill(완전 둥글게)
- 아이콘: 라인(블랙 스트로크) + 그린 도트
- 규칙: 둥글기 값은 위 세 가지로만 통일.

## 7. Components (컴포넌트)
- Primary 버튼: 배경 #B0C6E1, 흰 글자, pill, 높이 50px
- Ghost 버튼: 흰 배경, 1px #E4E4E8 보더, pill
- 카드: 배경 #F2F2F0, 모서리 22px, 호버 시 살짝 떠오름
- 통계: 큰 숫자 44px / 800 + 보조 라벨
- 네비게이션: 기본 font-weight 500, hover/active 시 700으로 변화

## 8. Do's and Don'ts (지킬 것·하지 말 것)
- 폰트 2개 이상 혼용 금지
- 카드 알록달록 채색 금지 (중립면 + 아이콘만 컬러)
- 그라데이션·과한 그림자·라이트그레이 선 금지
  - 예외: 사이드 내비의 스크롤 진행 표시줄(progress bar)은 블루 계열 그라데이션 허용 (얇은 기능 요소에 한함)
- 박스 좌측 컬러 스트라이프 금지
- 모션은 페이드업·카운트업 정도로 절제
