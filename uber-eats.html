Research

Every complaint traced back to one thing:
delivery mishaps.

Surveys and interviews made the pattern clear — frustration wasn't scattered. Every friction point users described led back to the same core problem: delivery mishaps, with no way to prevent them and no path to resolve them.

Core Problem
Delivery Mishaps

Users experience discomfort from delivery mishaps caused by poor communication and complex usability — and need better alternatives when things go wrong.

Key Insight

Users couldn't reach anyone —
and couldn't find what they needed.

Digging into the causes revealed a clear pattern: the app was built around completing orders, not around what happens when something goes wrong. Three precise UX failures defined exactly where the design had to step in.

Problem 01
Key information is hard to reach
Solutions
View all categories at a glance + detailed subcategories on home screen
Menu categorization instead of random recommendations
Final price always visible in a persistent floating bar — no unnecessary scrolling
Problem 02
No real alternative when a mishap occurs
Solutions
Location-based recommendations to prevent wrong-store orders
Home screen address mismatch alert before checkout
Contact channel for restaurant / driver to resolve errors fast
Problem 03
Editing feels unintuitive and risky
Solutions
More intuitive add / remove controls in the cart

Rather than a full redesign, each strategy closes exactly one structural gap — working within Uber Eats' existing design system.

UX Strategy 01

Catching address errors early,
and offering a fix if they slip through.

A home screen alert now catches address mismatches before browsing starts. If the user still reaches the cart with a farther store selected, a one-tap branch switch carries items over automatically.

01 · Address Mismatch Alert

Move the address warning from checkout to the home screen

GPS mismatch alert appears on the home screen before browsing starts — not at checkout after the cart is built
Out-of-range stores are visually dimmed, making wrong choices obvious at a glance
The original warning appeared only at checkout — after the cart was fully built. This redesign catches the error before any effort is wasted.
AS-IS

Warning fires at checkout — cart already built

TO-BE

Alert before browsing · out-of-range stores dimmed

02 · Nearest Branch Suggestion

One-tap switch to the nearest branch — cart items carry over

If the selected store isn't the closest one, an inline cart prompt offers a one-tap branch switch
Cart items transfer automatically — no rebuilding from scratch
Missed the nearest store? One tap switches it — without losing the cart.
UX Strategy 02

After delivery, users had no way to follow up.
Two new channels fix that.

Once delivery is marked complete, all contact options disappear. A 20-minute driver chat window and a 3-day restaurant review channel give users a direct way to follow up.

01 · Post-Delivery Driver Chat

20-minute driver contact window

A pop-up on the home screen lets users contact the driver for up to 20 minutes after delivery is marked complete
Order history moved to the bottom navigation — driver chat and "Get Help" are accessible together from each past order
Contact used to disappear the moment delivery completed. Now users have 20 minutes to resolve wrong-address or missing-item issues directly with the driver.
02 · Restaurant Review Channel

3-day review window with direct restaurant response

Users can write detailed reviews — with photos and menu recommendations — up to 3 days post-delivery
Restaurant owners receive reviews directly and can respond — creating a two-way communication channel that helps future users make informed choices
Reviews work when they feel like communication, not complaint forms. Restaurants can now respond, and future users benefit from the exchange.
UX Strategy 03

Hierarchical categories for faster browsing,
and separated cart controls for less confusion.

Instead of algorithm-picked recommendations, food is now organized into hierarchical categories for faster, intent-driven browsing. In the cart, three previously conflated actions are now separated into distinct, labeled controls.

01 · Category Navigation

Category-first home screen and a menu-focused restaurant detail page

Home screen: a category grid replaces the feed-emphasized layout. Tapping a category opens a results screen with main and subcategory tabs, so users narrow choices by intent, not algorithm.
Detail page: content-based tabs (Menu, Nutritional Info, Review) and subcategory chips replace algorithm-driven tabs
From the home screen to the detail page, every layer is now organized around what the user is looking for — not what the algorithm wants to surface.

02 · Cart & Checkout

Quantity control, deletion, and item editing separated into distinct, labeled actions

Trash icon in the stepper replaced with a minus button — a dedicated × handles item deletion, making quantity control and removal two clearly separate actions
Edit button surfaces item customization explicitly — no hidden interactions to discover
Floating checkout bar keeps the live total always visible — no scrolling to confirm the price
Each action — adjust, delete, edit — is now a distinct, labeled control. No guessing, no accidental taps.

Design System

Built on Uber Eats' own visual language.

Every screen stays within Uber Eats' existing system — Uber Move typeface, green primary palette — so the solutions feel native. Three principles guide the work: intuitive, reliable, hierarchical.

Reflection

The hardest part was resisting
the urge to redesign everything.

This project was about identifying exactly what needed to change and making those changes fit naturally within an existing system.

01
Surface complaints aren't the problem to solve

Delivery mishaps were the surface problem users described. Digging into the causes revealed three distinct gaps across prevention, recovery, and navigation, each requiring its own targeted solution.

02
Constraints sharpen decisions

Every solution had to integrate into existing screens without disrupting the overall flow. The address alert, the post-delivery chat window, and the category grid all had to feel like they always belonged in the app, not like additions layered on top.
